# Interview Talking Points

## Why I built this project

I wanted a practical Microsoft 365 lab project that demonstrates hands-on PowerShell scripting, Microsoft Graph usage, and read-only audit design.

## What the project does

- validates Microsoft Graph connectivity
- reads settings from a JSON configuration file
- retrieves user inventory
- retrieves sign-in summary
- retrieves group inventory
- retrieves disabled user inventory
- retrieves licensed user inventory
- retrieves authentication method summary
- generates an executive summary

## Why it is valuable

This project shows practical Microsoft 365 administration skills, reporting design, modular scripting, documentation discipline, and GitHub publishing.

## Key design choice

The toolkit is intentionally read-only, so it can support audit and validation scenarios without making production changes.
